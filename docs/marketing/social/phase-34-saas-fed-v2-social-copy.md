## Post 2 — What federation means in practice

If you're running one org, SaaS Fed v2 doesn't require action — it runs in the background.

If you're running multiple orgs, federation is what makes multi-org setups reliable: org boundary enforcement, heartbeat reliability, and cleaner org lifecycle management.

---

## Post 3 — What's new in v2

SaaS Federation v2 brings:
- **Improved cross-tenant isolation** — cleaner org boundary enforcement
- **Cleaner org lifecycle management** — provisioning and teardown under better control
- **Tighter alignment with Partner API Keys** — the org management API works more reliably with the `mol_pk_*` provisioning flow
- **Better external workspace discovery** — agents on laptops or other networks benefit from improved heartbeat reliability

---

## Post 4 — External workspaces

If you have agents running on laptops or other networks outside the platform — external workspaces — v2 improves their discovery and heartbeat reliability. Agents stay registered and reachable more consistently.
