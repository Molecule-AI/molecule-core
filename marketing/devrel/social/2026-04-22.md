### Post 1 — Hook

Most Discord bot integrations require:
→ Developer Portal setup
→ OAuth flow
→ Gateway connection
→ Intent configuration

That's overhead before your agent can say hello.

Molecule AI's Discord adapter requires:
→ One webhook URL

That's it.

---

### Post 2 — How it works

The Discord adapter uses two standard primitives:

→ Incoming Webhooks for outbound messages (agent → Discord)
→ Discord Interactions for inbound slash commands (Discord → agent)

No polling. No Gateway. No OAuth.

The agent initiates outbound connections only — it works behind NAT without requiring an inbound-reachable endpoint. Discord calls the platform endpoint, the platform routes to the workspace via A2A, the agent responds, the response goes back through the webhook.

Discord → Platform → Workspace via A2A → Discord

---

### Post 3 — What you get

Type `/ask what's our deployment status?` in any Discord channel.

The adapter parses the command, delivers plain text to a Community Manager workspace, the workspace delegates to the right sub-agent, and the answer comes back to the channel.

The routing is invisible to the user. Your whole agent team, accessible from a Discord server your team already lives in.
